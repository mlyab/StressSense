# StressSense — 5-minute presentation outline

11 slides, ~25–30 s each. Speaker notes in italics. Numbers are from
`results/metrics.json` and `results/error_analysis.txt`.

---

### 1. Title / Motivation
- **StressSense: Wearable Stress Detection under Physical Activity Confounding**
- Malya Aboun — UCSC Deep Learning
- *Wearables claim to detect stress from physiology. But exercise raises the same
  signals. Can a model tell them apart?*

### 2. Problem
- Detect stress from Empatica E4 signals — but exercise mimics stress (↑HR, ↑EDA).
- Framed as **4-class window classification**: baseline / stress / aerobic / anaerobic.
- *The real question isn't accuracy — it's whether stress gets confused with exercise.*

### 3. Dataset
- PhysioNet wearable dataset: 36 subjects, 3 session types.
- Signals: EDA, skin temp, HR, accelerometer (+ BVP, IBI). Different sample rates.
- Labels from session type + protocol tags (mapping taken from the dataset's own notebook).
- *Handled real messiness: covered sensors, dropped connections, split sessions.*

### 4. Pipeline
- raw E4 CSVs → resample to a common 4 Hz grid → 30 s windows → normalize → 1D CNN.
- One command each: `dataset.py → train.py → evaluate.py → demo.py`.
- *Fully config-driven and seeded; reproducible end to end.*

### 5. Model
- Compact **1D CNN**: 3 conv blocks (32/64/128) → global avg pool → linear head (~90k params).
- 4 channels: EDA, TEMP, HR, **ACC motion intensity** (gravity removed — pure movement).
- BVP dropped on purpose: can't downsample a 64 Hz pulse to 4 Hz; use derived HR.
- *Local, shape-based cues → a small CNN is the right tool, not a big sequence model.*

### 6. Training
- Adam + class-weighted cross-entropy (baseline/stress are minority).
- **Subject-level split** (25/5/6) — no person leaks across splits.
- Best model by validation macro-F1; ~2 min on CPU.
- *The split is the most important methodological choice — it tests new people.*

### 7. Evaluation
- Metrics: accuracy, macro-F1, per-class P/R/F1, confusion matrix, computed error analysis.
- Macro-F1 is the headline (imbalanced classes).
- *Everything auto-generated by evaluate.py — no hand-edited numbers.*

### 8. Demo
- Gradio app: pick/upload a window → predicted class + confidences + signal plot.
- Live: classify a stress window, then an exercise window.
- *Show the model is right for understandable reasons (the motion trace).*

### 9. Results & confusion analysis
- Test: **accuracy 0.54, macro-F1 0.53** (6 held-out subjects).
- **Stress vs. physical activity: 0% confusion both ways** — the main result.
- Hard cases: aerobic↔anaerobic ~30%; stress under-detected (recall 0.33, precision 0.76).
- Easiest: aerobic (F1 0.62). Hardest: stress (F1 0.46).
- *The win: stress is never mistaken for exercise. The honest gap: subtle stress vs. rest.*

### 10. Limitations & future work
- Few subjects (test = 6, high variance); lab "stress" ≠ clinical stress; session confounding.
- **Not a medical/diagnostic tool.**
- Next: BVP/HRV branch, subject-wise CV, per-subject normalization, regress stress level.

### 11. Lessons learned
- Subject-level splitting changes the story (val 0.64 → test 0.53).
- The confusion matrix is more informative than accuracy.
- Honest preprocessing choices (motion intensity, dropping BVP) beat using every signal blindly.
- Handling real-world data defects is most of the work.
